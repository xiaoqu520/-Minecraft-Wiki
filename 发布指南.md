## 若导入后侧栏很乱、全是英文（必读）

你之前看到的现象（首页标题叫 **gitbook**、链接带 `/gitbook/`、侧栏是 `how to pay`、`daily signin` 等）是因为：

1. **未指定文档根目录** — GitBook 把整个仓库当文档，把 `gitbook` 文件夹当成一个普通目录，**没有读取** `gitbook/SUMMARY.md`。  
2. **侧栏标题来自文件名** — 未配置 `title` 时会把 `how-to-pay` 转成英文标题。

**已在本仓库修复：**

| 文件 | 作用 |
|------|------|
| 仓库根目录 [`.gitbook.yaml`](../.gitbook.yaml) | 声明 `root: ./gitbook/`，让 GitBook 只同步文档目录 |
| [`SUMMARY.md`](SUMMARY.md) | 中文分组目录（`## 开始游戏` 等） |
| 各页 YAML `title:` | 侧栏显示中文，如「充值与支付步骤」 |
| 部分中文文件名 | 如 `充值与支付.md`、`赞助条例.md` |

**你需要做的：**

1. 将本次更新 **push 到 GitBook 绑定的 Git 仓库**。  
2. 打开 [GitBook Space 设置](https://app.gitbook.com/) → **Integrations → Git Sync**：  
   - **Project directory / Content root** 留空或填 `/`（由 `.gitbook.yaml` 的 `root` 指向 `gitbook/`）。  
   - **不要** 只填 `gitbook` 同时又让仓库根缺少 `.gitbook.yaml`。  
3. 在 Space **Settings → General** 把 Space 名称改为 **安途服务器**（不要叫 gitbook）。  
4. 点击 **Sync now** 等待重建。  
5. 在 GitBook 侧栏把 **维护** 分组下的「发布指南」「配图清单」设为 **Hidden**（可选）。

同步成功后，首页应为 **安途服务器 · 玩家手册**，侧栏按 **一至九** 中文章节显示。

## 三级侧栏验收（本次整理后）

Push 后在新 Space 中确认侧栏可 **展开 3～4 层**，例如：

| 层级 | 示例 |
|------|------|
| 一级 `##` | 四、玩法系统 |
| 二级 | 玩法系统总览 → 日常与成长 |
| 三级 | 每日签到、每日任务… |
| 四级（仅会员） | 会员档位导览 → VIP |

若子页未缩进，检查 [`SUMMARY.md`](SUMMARY.md) 是否使用 **2 空格** 递增缩进（勿用 Tab）。

维护分组 **九、维护** 建议在 GitBook UI 设为 **Hidden**。

## 侧栏「只有一层」时（如点券与充值）

若 **七、点券与充值** 下只显示一个「点券充值说明」、看不到子页：

1. 确认已 **Sync now**，且仓库含最新 [`SUMMARY.md`](SUMMARY.md)（章首页应为 **赞助指南**，下挂 5 个子页）。  
2. 旧版 [`donation/README.md`](donation/README.md) **已移出侧栏**，避免与赞助指南重复。  
3. **五、经济系统** 现为 4 篇 **平铺**（无「货币体系」空壳层）。  
4. **二、新手** 中「注册与登录」为 **二级直达**（已删除仅 1 页的「账号与安全」hub）。

## 内容真实性（维护必读）

玩家手册中的 **命令、价格、数量、权限** 应以本仓库旁路目录 **`server/plugins`** 内配置文件为准，禁止凭印象编写。

| 允许 | 禁止 |
|------|------|
| 摘录 TrMenu `command` / `lore`、CooPay `rewards.yml`、Residence `groups.yml` 等 | 编造未在配置出现的指令或价格 |
| 标注 **制作中 / 暂未开放**（如主菜单 F 格、地皮 NPC） | 为未上线功能写收益或操作步骤 |
| 抽奖写「以游戏内预览为准」 | 虚构宝箱权重表（除非与预览一致） |

改文档后建议：对照游戏内实测一轮，再 **Sync now**。

## 方式 A：Git 同步（推荐）

1. 将 **整个** `c:\服务器` 仓库 push 到 GitHub（含根目录 `.gitbook.yaml` 与 `gitbook/`）。  
2. GitBook → **Integrations** → **GitHub** → 选择该仓库。  
3. **不要** 手动把 Content root 设成 `gitbook` 而省略根目录的 `.gitbook.yaml`；推荐仓库根 + `.gitbook.yaml` 自动识别。  
4. 推送后自动构建，首页为 `README.md`。

### 检查清单

- [ ] 仓库根存在 [`.gitbook.yaml`](../.gitbook.yaml)，且 `root: ./gitbook/`  
- [ ] `SUMMARY.md` 含 **九个大类** 与玩法 **5 个子类导览**（`gameplay/hubs/`）  
- [ ] **七、点券** 展开为 **赞助指南 + 5 子页**（非单独「点券充值说明」）  
- [ ] **五、经济** 为 4 篇平铺子页  
- [ ] 侧栏可展开至 **每日签到** 等三级页面  
- [ ] 会员可展开至 **VIP** 四级页面  
- [ ] 图片已上传或仍保留 `[待补图]`（见 [配图清单](配图清单.md)）  
- [ ] **九、维护** 已 Hidden（可选）

## 方式 B：导入 Markdown

1. GitBook → **Import** → **Markdown** / **Upload folder**。  
2. 上传整个 `gitbook` 文件夹。  
3. 手动确认目录层级与 `SUMMARY.md` 一致（部分导入需重新拖拽排序）。

适合快速预览：在 GitBook 中新建页面，将各 `.md` 正文粘贴进去。工作量大，仅作临时方案。

## 常见问题

**Q：SUMMARY 不显示子页面？**  
A：检查缩进为空格 2 或 4 一致，链接为相对路径 `folder/page.md`。

**Q：中文乱码？**  
A：确保文件 UTF-8 编码保存。

**Q：图片不显示？**  
A: GitBook 需使用其图床或上传资产；本地路径 `![](...)` 需改为 GitBook 上传后的 URL。
